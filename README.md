# Project Description

## Structure of project
This project is an Alzheimer's disease (AD) risk analysis.

- `Source/` contains all code to generate tables, figures and report
  - `Source/00_restore_packages.R` use package `renv` to provide the packages version used
  - `Source/01_table1.R` produces Table 1
  - `Source/02_figure1.R` produces Figure 1
  - `Source/03_render_report.R` produces the report
- `Reports/` contains Rmd and html version of report
- `Output/` contains Table 1 and Figure 1 generated
- `Data/` contains the data about AD risk
- `.Rprofile`, `renv.lock` and `renv/` include information about environment
- `Makefile` contains all the make commends to generate report automatically
- `Dockerfile` contains the code to generate a container to run the report

## About the report
The report include 3 parts:
- Introduction of Data
- Table 1 for Demographic Variables
- Figure 1 for Mini-Mental State Examination score and AD Risk.

## Synchronize R packages via `renv`
The environment information is provided by `renv`, ahead of `make` report, please run `Source/00_restore_packages.R` or `make install` to restore the packages used.

The restore is successful if there is output:
`- The library is already synchronized with the lockfile.`

## Build and run of project image

The docker container for this project be generated by `make project_image`, please note this may take time.
The container is also available on DockerHub at [ytliu36/final_project_image](https://hub.docker.com/repository/docker/ytliu36/final_project_image/general): , you can `docker pull ytliu36/final_project_image:latest` to retrieve it.

After get the imgae either my `make` or `docker pull` you can generate a report in `report/DATA550_final_Yutong.html`.
- If you make a image by `make`, you can use `make run_report` for MacOS/Linux or `make run_report_win` fro Windows to generate report.
- If you make a image by `docker pull`, you can use `make run_report_hub` for MacOS/Linux or `make run_report_win_hub` fro Windows to generate report.
